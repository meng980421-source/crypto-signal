# 交易策略说明书 — 区间交易 + 缠论 + 六爻(梅花)

## 0. 总原则
- 标的:OKX 上的 BTC/USDT、ETH/USDT 永续合约。
- **持仓周期:1–5 天(波段)**;信号用于推送通知、手动下单,不自动执行。
- **信号判断周期:每 15 分钟检查一次**(注意:判断周期 ≠ 持仓周期)。
- 三层信号**各自独立、可单独开关**(`range_on` / `chan_on` / `liuyao_on`),便于单独测、组合测,衡量每层边际贡献。
- 默认参数都是**起点假设**,必须回测调优 + 样本外验证后才可信。

## 1. 数据与周期(多周期)
- **15m**:信号检查与入场触发的主周期(每根 15m 收盘判断一次)。
- **1h**:定义区间(支撑/阻力)、震荡过滤、缠论中枢/背驰。
- **1d**:大环境参考。
- 指标按收盘价计算;通过 ccxt 拉 OKX 公共行情(无需 API key)。

## 2. 第一层:区间交易(核心引擎)
### 2.1 区间识别(在 1h 上)
- 回看窗口 `N = 48` 根 1h(约 2 天)。上沿(阻力)=窗口最高价;下沿(支撑)=窗口最低价;中线=两者均值。
- **震荡确认**:1h ADX(14) `< 25` 才做区间;`≥ 25` 视为趋势,**暂停开新仓**。
### 2.2 入场(在 15m 上触发)
- **做多**:15m 收盘价 ≤ 下沿 × (1 + `edge_buffer`),`edge_buffer = 0.5%`。
- **做空**:15m 收盘价 ≥ 上沿 × (1 − `edge_buffer`)。
- 同一币种同方向同时只持 1 仓,持仓中不重复发同向信号。
### 2.3 出场(任一先到即平)
- **止盈**:中线(默认)或对侧边沿。
- **止损**:15m 收盘跌破下沿 × (1 − `stop_buffer`) / 升破上沿 × (1 + `stop_buffer`),`stop_buffer = 1%`。**强制止损,不可取消**。
- **时间止损**:持仓满 5 天仍未触发,强制平仓。
### 2.4 仓位
- 每笔风险 `risk_per_trade = 1%` 权益;仓位 = 权益 × 1% ÷ |入场价 − 止损价|;杠杆 ≤ **3×**,不主动加杠杆。

## 3. 第二层:缠论确认(可选,在 1h 上)
- 1h 上算 分型 → 笔 → 中枢 → 背驰(用现成库 `czsc` 或 `chan.py`,不要从零写)。
- **确认**:做多需下沿附近**底背驰**;做空需**顶背驰**。
- **校准**:存在 1h 中枢时,用中枢上下沿替代 2.1 的高低点。
- 开关 `chan_on`。

## 4. 第三层:六爻(梅花·时间起卦 + 体用生克,可选)
> 本层只依赖时间、不看价格,本质是日历型信号。务必样本外验证,警惕过拟合。
### 4.1 起卦(每根 15m 收盘时刻)
1. 取**年支数**(子1…亥12)、**农历月**、**农历日**、**时辰支数**(子1…亥12);用 `lunar_python` 或 `sxtwl` 转换。
2. 八卦先天数:乾1 兑2 离3 震4 巽5 坎6 艮7 坤8。
3. **上卦** = (年支 + 月 + 日) mod 8(余 0 取 8)。
4. **下卦** = (年支 + 月 + 日 + 时支) mod 8(余 0 取 8)。
5. **动爻** = (年支 + 月 + 日 + 时支) mod 6(余 0 取 6)。
6. 本卦 = 上卦(外)+ 下卦(内);动爻在 1–3 爻则**下卦为用、上卦为体**,在 4–6 爻则反之。
### 4.2 五行生克
- 乾/兑=金,震/巽=木,坤/艮=土,离=火,坎=水。
- 生:木→火→土→金→水→木;克:木→土→水→火→金→木。
### 4.3 体用 → 方向
- **用生体** 或 **比和** → 偏多;**用克体** → 偏空;**体生用** 或 **体克用** → 观望。
- 体克用是否改算偏多 → 可调项 `tikeyong_bullish`(默认 False)。
- 开关 `liuyao_on`。

## 5. 三层组合规则
- **区间层**:决定在哪、做多还是做空(+ 止损止盈仓位)。
- **缠论层**(开启时):无对应背驰则不发信号。
- **六爻层**(开启时):方向需与候选信号一致,相反则观望不发。
- 三层全开:候选(区间)→ 缠论背驰确认 → 六爻方向不冲突 → 发出买/卖通知。

## 6. 通知内容(实时推送)
- **入场通知**:币种、方向(买/卖)、建议入场价、**止损价**、止盈目标。
- **离场通知**:价格触及**止盈或止损**时,推送提醒平仓。
- 渠道:微信(Server酱 / PushPlus)。

## 7. 回测要求
- 计入成本:OKX taker ≈ 0.05%、资金费(每 8h)、滑点(1–2 个最小跳动)。
- **样本外验证**:历史切「调参段 / 检验段」,只在调参段优化,检验段评估;六爻层尤其要看是否只是调参段的偶然。
- **消融测试**:①区间单独 ②区间+缠论 ③区间+六爻 ④三层全开,对比指标。
- 指标:胜率、期望值、盈亏比、最大回撤、Sharpe、交易次数、总收益率。
- 牢记:**高胜率 ≠ 赚钱**,必看期望值与最大回撤。

## 8. 参数表
| 参数 | 默认 | 含义 |
|------|------|------|
| `eval_tf` | 15m | 信号判断周期 |
| `range_tf` | 1h | 区间 / 缠论周期 |
| `N` | 48 | 区间回看(1h 根数) |
| `adx_len` / `adx_thresh` | 14 / 25 | 震荡判定 |
| `edge_buffer` | 0.5% | 入场贴边容差 |
| `stop_buffer` | 1% | 突破止损容差 |
| `tp_mode` | 中线 | 止盈位置 |
| `max_hold` | 5 天 | 时间止损 |
| `risk_per_trade` | 1% | 单笔风险 |
| `max_leverage` | 3× | 杠杆上限 |
| `chan_on` / `liuyao_on` | True | 层开关 |
| `tikeyong_bullish` | False | 体克用是否偏多 |

## 9. 声明
参数是起点,不是「能赚钱的配方」。六爻层无已知预测机制,纳入只为用同一套回测标准客观检验;是否保留以**样本外回测结果**为准。本文件仅为技术实现规格,不构成投资建议。
